Это учебный проект начального уровня, исключаем сложные зависимости и оверинжиниринг
Цель: собрать первый фриланс-уровня MVP, который демонстрирует базовые компетенции вайбкодера:
-работа с API (в виде webhook/endpoint);
-работа с базой данных (SQLite);
-базовая надёжность (валидация, обработка ошибок, логирование);

Симуляция клиента: небольшой бизнес (услуги / онлайн-школа / магазин
Проблема: заявки приходят из формы / соцсетей / таблицы «как попало», часть теряется, менеджер узнаёт поздно.
Запрос: «Сделайте так, чтобы любая новая заявка автоматически попадала в базу, и мы видели, что она пришла».

Проект MVP:
мини-сервис, который:

1) Принимает заявку через webhook
HTTP endpoint POST /lead
Формат входных данных (пример JSON):
{
  "name": "Ирина",
  "contact": "+79990000000",
  "source": "landing",
  "comment": "Хочу консультацию по тарифам"
}

2) Сохраняет заявку в SQLite
Таблица leads с полями:
id (автоинкремент)
created_at
name
contact
source
comment

3) Фиксирует событие «заявка принята»
3А Event Log — запись в файл events.log (или таблицу events) с текстом New lead saved: <id>.
3Б  Email-уведомление — отправка письма менеджеру «Новая заявка: …».

4) Обрабатывает ошибки
если JSON невалидный / отсутствует contact → возвращать понятную ошибку (HTTP 400)
если база недоступна → ошибка (HTTP 500) + запись в лог

🛠 Стек технологий (рекомендуемый)
Web-фреймворк: FastAPI
База данных:SQLite
Прочее: Python 3.10+
pydantic для валидации
smtplib или внешний email-провайдер для отправки email

Ограничения:
Один endpoint POST /lead
Одна база SQLite
Один сценарий «новая заявка»
Без деплоя на VPS (локально)
Простая дедупликация
