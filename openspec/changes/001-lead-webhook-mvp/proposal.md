# 001 MVP webhook для заявок

## Краткое описание

Нужно собрать учебный FastAPI-сервис начального уровня, который принимает одну
заявку через webhook, сохраняет принятую заявку в SQLite, фиксирует
операционные события и отправляет email-уведомление через SMTP-настройки из
`.env`.

## Проблема

Небольшие бизнесы получают заявки из форм, соцсетей и таблиц. Часть заявок
теряется или замечается слишком поздно. MVP должен дать один надежный локальный
endpoint, который сохраняет каждую новую заявку и оставляет понятный след, что
заявка была получена.

## Цели

- Предоставить `POST /lead` для входящих JSON-заявок.
- Валидировать входные данные и возвращать понятные HTTP 400 ошибки для
  невалидного JSON или отсутствующего/пустого поля `contact`.
- Сохранять заявки в локальную SQLite-базу.
- Выполнять дедупликацию по `name` в нижнем регистре и нормализованному
  российскому телефону в формате `+7XXXXXXXXXX`.
- Записывать `New lead saved: <id>.` в `events.log` после сохранения новой
  заявки.
- Отправлять SMTP email-уведомление с настройками из `.env` и также
  фиксировать результат отправки в логе.
- Возвращать HTTP 500 и логировать ошибку, если SQLite недоступна.
- Подготовить 5-10 ручных тестовых payload'ов и инструкции в README.

## Не входит в задачу

- Деплой на VPS.
- Админ-панель.
- Фоновый worker или внешняя очередь.
- Сложная интеграция с CRM.

## Выбранный стек

- FastAPI
- SQLite через стандартный Python-модуль `sqlite3`
- Pydantic-валидация через FastAPI
- `smtplib` для email-уведомлений
- `pytest`, `httpx` и `ruff` для проверки
